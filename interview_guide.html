Medsteps Rebuild Interview Guide

Comprehensive Guide: Expectation, Concept, Code Explanations & Technical Architecture

💡 Key Strategy: How to Present as a Fresh Graduate/Entry-level Candidate

  • Confidence over jargon: If you don't know a complex term, explain the logic. Interviewers look for clear problem-solving minds.
  • Focus on Clean Code: Emphasize that you separated features into clean, modular files inside the src/components directory to keep the project readable.
  • Mention performance: Highlight the use of Tailwind CSS v4 for lighter bundles and inline SVGs instead of unnecessary third-party icon packages.
Category 1: Project Architecture & Code Structure

Q1 Explain the directory structure of this project. Why did you choose it?

"I structured this project using a standard component-driven architecture. The core configurations (Vite config, package.json, Tailwind imports) sit in the root. Inside the src/assets folder, I stored the local image assets. The reusable UI section files like Navbar, Hero, Features, Consultants, and Footer are separated into the src/components/ folder. The main App.jsx serves as the parent wrapper that hosts and orchestrates these components."
Interviewer Expectation: They want to see if you understand modular code. Separating sections into components proves that you know how to build maintainable, readable React projects.

Q2 What is the advantage of using Vite over Create React App (CRA)?

"Vite is significantly faster than Create React App. CRA uses Webpack, which bundles the entire application before starting the dev server. Vite uses native ES Modules (ESM) in the browser to deliver file changes on-demand, which makes development starts and Hot Module Replacement (HMR) instantaneous, regardless of the project size."
Interviewer Expectation: Showing that you know modern bundling tools. Webpack is legacy; Vite is the modern industry standard.
Category 2: Core React Concepts

Q3 How does the Light/Dark mode state work in this application?

"In App.jsx, I defined a state variable: const [darkMode, setDarkMode] = useState(false). I passed this state and its setter function down to the Navbar component as props. Inside App.jsx, a useEffect hook monitors the darkMode state. Whenever it toggles, it modifies the class list of the HTML root document element (document.documentElement). Adding the .dark class allows Tailwind to trigger all our dark: variant style rules."
Interviewer Expectation: A very common React 19 test. It verifies if you know state sharing, component props, and how React interacts with the global document Object Model (DOM) using effects.

Q4 What is the difference between props and state in React?

"State represents the internal, mutable data managed by a component itself (like the isOpen menu toggle in Navbar). Props (short for properties) are read-only variables passed down from a parent component to a child component (like passing darkMode to Navbar) to share configuration."
Interviewer Expectation: Junior-level absolute fundamental check. You must explain that props are immutable (read-only) while state is local and mutable.

Q5 Why did you use functional components and hooks instead of class components?

"Functional components combined with React Hooks are the modern standard. They make components lighter, reduce boilerplate code (no constructor or complex binding of 'this'), and allow developers to easily reuse stateful logic without changing the component hierarchy."
Interviewer Expectation: Verifying if your knowledge is up-to-date. Class components are legacy (pre-2019). Functional components are standard.
Category 3: Tailwind CSS & Responsive Styling

Q6 How does Tailwind CSS v4 differ in configuration compared to v3?

"In Tailwind v3, configurations were handled in a JavaScript file (tailwind.config.js). In Tailwind v4, the configuration is CSS-first. We import Tailwind directly in index.css using @import 'tailwindcss'; and customize the theme variables (like fonts, custom colors, and custom variants) directly inside a CSS block called @theme { ... }. This eliminates the need for separate JS configuration files."
Interviewer Expectation: Shows that you keep up with the latest web standards. Tailwind v4 is the newest release and makes configurations much cleaner.

Q7 How did you implement responsive design in the Bento Grid layout?

"I used Tailwind's mobile-first breakpoints. The Bento Grid in Features.jsx uses: grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6. On mobile screens, it defaults to one single column. On tablets (medium screens at 768px), it switches to 2 columns (md:grid-cols-2). On laptops/desktops (large screens at 1024px), it expands to 3 columns (lg:grid-cols-3). I also used md:col-span-2 on the dashboard card to make it dynamically stretch wider on larger screens."
Interviewer Expectation: Tests your practical CSS Grid and flexbox layout skills. Responsive layout grids are crucial for modern web development.

Q8 Why did you use inline SVGs for social media icons instead of importing them from Lucide React?

"Lucide React recently deprecated and removed brand logos (like Facebook, Twitter, and LinkedIn) to avoid trademark conflicts and reduce compile sizes. To solve this without adding heavy third-party packages, I wrote lightweight, custom inline SVG elements. They are fully responsive and adapt to hover effects instantly using the Tailwind class fill-current."
Interviewer Expectation: A brilliant answer that demonstrates troubleshooting skills, performance awareness, and understanding of dependency management.
Category 4: Practical Code Explanations

Q9 Walk me through the implementation of the Mobile Toggle Menu in the Navbar.

"In Navbar.jsx, I set up a state: const [isOpen, setIsOpen] = useState(false). On mobile view, I render a hamburger button that toggles `isOpen` on click. Below the header, I write conditional rendering logic: {isOpen && ( ... )}. When `isOpen` is true, the drawer container opens, showing the navigation links in a vertical stack. Clicking any link closes the drawer automatically."
Interviewer Expectation: Tests if you can explain interactive components line-by-line. Keep your answer simple and follow the state-to-DOM logic.

Q10 In Consultants.jsx, how did you render the consultant list?

"I created an array of objects called `consultants` containing the doctor's details (name, specialty, rating, reviews count, and image source). In the JSX template, I used the JavaScript `.map()` method to loop over this array. For each doctor, it renders a styled card layout with unique keys (key={doctor.id}). This makes the code highly dry (Don't Repeat Yourself) and easy to scale if we fetch the doctors list from an API later."
Interviewer Expectation: Map loops are fundamental in React. Always mention the use of the `key` prop, explaining that it helps React's virtual DOM track which items changed, added, or removed.
